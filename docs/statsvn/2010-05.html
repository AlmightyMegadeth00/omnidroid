case.andrew 2010-05-31 19:32 #16 Rev.: 725

I think it makes sense for use to add all the permissions that we think
Omnidroid will want to implement in the future as well. Specifically
for the auto-update feature now added in Android 2.2, the feature will
only auto-update if the permissions haven't been changed from release to
release. So including permissions now avoids users from having to
re-accept with each version where we've added a permission.

I've added all the permissions I think might apply to Omnidroid in the
future and sorted them alphabetically. Also, since one of these
permissions fixes an active bug (incoming call events), I've bumped the
version so I can update the Market application.

Refactoring the package name from edu.nyu.cs.omnidroid to edu.nyu.cs.omnidroid.app. This is a result of the original keystore used to sign the market app being lost or broken. Unfortunately the market doesn't have a way to handle the situation where the key is lost/corrupted. And although you can unpublish an app, you can't delete it or replace it. So we had to rename the package instead. :-/

From now on I'll be the only one posting the app on the Market to avoid this happening again.
