Q: What is OmniDroid all about?
A:  OmniDroid is an event/action manager that allows users to automate system functionality based on incoming application intents. Applications will be able to plug-in configurations to allow our application (and therefore others) to interact with them.
Q: What is the motivation of this project? 
A:    - Flexibility between app communication
       - Automate event handling
       - Not on iphone/other mobile platforms
       - Generic communication platform --> apps need to be omni aware.
Q: What can OmniDroid do?
A:  It connects the events of one application with actions of another application.
Some of the things that you can do with OmniDroid are:
  • 1st event of the day -> Alarm Wake up
  • Birthday -> Toast Action
  • Cold/Rainy Weather -> Toast Msg/Suggest Clothing
  • Toast Action -> Email/SMS/Call
  • Date/Time -> Suggest Meals
  • Email Recd -> Organize/respond based on status
  • Location -> Set phone profile
  • Silent mode/call -> Send SMS
  • Set Mood -> Choose Music Playlist
  • Update profile on one app -> Update profile on others
  • New song playing -> update away msg
  • Specific song plays -> Picture Display
  • Get email from a contact -> Queue song associated with them
  • Locale coordinate changes -> BooRah (Restaurant listings and reviews)
  • Alert on Stock Ticker -> Send email to Broker
  • Update "Not Well" Status -> Send notification to Bosses/Friends
  • WiFi network -> Automatically connect
  • If location is changing rapidly -> Activate GPS logging to keep track of speed/path/etc, send email or publish somewhere.
Q: Who/Why should one use OmniDroid?
A:     - Right now: Developers who like to build on existing frameworks
        - Soon: Anyone who wants to automate event handling
