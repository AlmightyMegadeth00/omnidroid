Introduction

 

OmniDroid will allow the users to automate the communication channel between applications. The Event/Actions will be based on the Incoming/Outgoing intents from the OmniDroid application. This application will provide support for the built-in applications. It will also allow other applications to plugâ€“in configurations that will enable them to communicate with the system more effectively.

Need for OmniDroid

The idea behind OmniDroid is to provide the user with a UI to manage the actions when certain events are triggered. The current communication mechanism between the applications does not allow them to customize the actions based on certain events. Example: If a user adds a contact on his/her social networking application then that application could fire off an event notification letting us know that an event has occurred and OmniDroid could allow the user to pass this information to other applications that could use this information like adding this contact to our address book.
